anyREST

anyREST is a “fake” rest API you can use as a mock back-end during your prototyping. It’s endpoints accept any CRUD request, and will store any POSTed data for up to 7 days of inactivity.

Here’s how it works.

  1. The first this to do is head over to anyest.com/new and generate a new unique base URL, It will look something like this:
https://anyrest.com/v1/[unique token]
  1. Now that you have this base URI you can send any of the following requests:
  1. GET, to retrieve the data stored in the specified location, can be an object or list of objects.
  1. POST, to create a new object at the provided location
  1. PUT, to fully update the object at the specified location given it’s id
  1. PATCH, to partially update the object at the specified location given it’s id
  1. DELETE, to deleted the object at the specified location given it’s id
  1. All HTTP methods support some general parameters, such as sorting, filtering and prettifying the output.

Here are some specifics on each method:

GET

  • If the specified locations contains objects who’s keys are all integers, it will return a list of their IDs.
  • Otherwise, it will return the object stored at the provided location.
  • You can pass the following parameters:
    • ?fields=feild1[,feild2,…] only include the listed fields in the output.
    • ?sort=[-]field to sort based on a certain field, add a negative to go in decreasing order.
    • ?[field]=value to filter on a certain field.

POST

  • If the specified locations contains objects who’s keys are all integers, it will insert the provided object at the location, giving it the next highest integer as an ID. If the path contains levels that do not exist, they will be created.

PUT

  • Will replace the object at the specified location, no restrictions. NOTE: This means you can replace entire collections with objects.

PATCH

  • Will update the object at the specified location in the following ways:
    • Will add any fields that are not in the stored object
    • Will replace any existing fields. NOTE: this does not work for nested fields.

DELETE

  • Will set the object at the specified location to null.

General parameters

  • You can pass ?pretty=true with any method to get the output printed as pretty json.

Example

  • POST : /v1/[token]/tickets/0/members with body {”name”: “Samuel”}

The data will be stored as follows (nested levels are automatically created.

{
	"tickets": {
		"0": {
			"members": {
				"0": {
					"name": "Samuel"
				}
			}
		}
}
  • POST : /v1/[token]/tickets with body {}

Data will be stored as follows

{
	"tickets": {
		"0": {
			"members": {
				"0": {
					"name": "Samuel"
				}
			}
		},
		"1": {
		}
}
  • GET : /v1/[token]/tickets will return:
[0,1]
  • GET : /v1/[token]/tickets/0 will return:
{
	"members": {
		"0": {
			"name": "Samuel"
		}
	}
}
  • PUT : /v1/[token]/tickets/0 with body {”show”: “blink-182”}

Data will be stored as follows

{
	"tickets": {
		"0": {
			"show": "blink-182"
			}
		},
		"1": {
		}
}
  • PUT : /v1/[token]/tickets/0 with body {”year": 2022}

Data will be stored as follows

{
	"tickets": {
		"0": {
			"show": "blink-182",
			"year": 2022
			}
		},
		"1": {
		}
}
  • DELETE : /v1/[token]/tickets/0
{
	"tickets": {
		"0": null,
		"1": {
		}
}
